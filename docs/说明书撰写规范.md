# Patent2Pic 说明书撰写规范

> 版本：V1.0 | 更新日期：2026-05-26

---

## 1 规范目的与适用范围

### 1.1 规范目的

本规范旨在为 Patent2Pic 用户使用说明书的撰写与维护建立统一标准，确保：

- 各版本说明书在结构、风格、排版上保持一致性
- 内容面向用户使用，聚焦功能操作，避免技术实现细节
- 图示表达规范、清晰，有效辅助文字说明
- 版本迭代时更新高效、差异可追溯

### 1.2 适用范围

本规范适用于 Patent2Pic 桌面应用用户使用说明书的撰写、修订与审核全过程。所有参与说明书编写的人员均应遵循本规范。

### 1.3 版本更新规则

| 更新类型 | 触发条件 | 更新要求 |
|---------|---------|---------|
| 小版本更新 | 功能微调、界面优化、Bug 修复 | 更新受影响章节内容，补充版本更新清单 |
| 大版本更新 | 新增核心功能、界面重大改版、架构调整 | 全面审查并更新所有章节，重新绘制受影响的图示 |
| 紧急修正 | 说明书内容错误或遗漏 | 仅修正错误部分，记录修正日志 |

> **规则 1.1**：每次说明书更新必须在文档末尾的"版本更新清单"中记录变更内容、变更日期和变更人。

> **规则 1.2**：**HTML 与 PDF 同步更新**。每次更新说明书时，必须同时更新 HTML 版本（`docs/user-manual.html`）和 PDF 版本。PDF 文件命名为 `Patent2Pic-用户使用说明书-VX.X.X.pdf`，版本号与 HTML 一致。生成新版本 PDF 后，需删除旧版本 PDF 文件。

### 1.4 PDF 生成方法

使用 wkhtmltopdf 工具生成 PDF。

**前置条件**：系统需安装中文字体，否则 PDF 中中文将无法正常显示。

```bash
# 安装中文字体（Ubuntu/Debian）
apt-get install -y fonts-noto-cjk fonts-noto-cjk-extra
fc-cache -f -v
```

生成 PDF 命令：

```bash
cd /workspace/docs
wkhtmltopdf \
  --enable-local-file-access \
  --page-size A4 \
  --margin-top 25mm \
  --margin-bottom 25mm \
  --margin-left 20mm \
  --margin-right 20mm \
  --encoding UTF-8 \
  user-manual.html \
  Patent2Pic-用户使用说明书-VX.X.X.pdf
```

## 2 文档结构规范

### 2.1 必备章节

每版说明书必须包含以下章节，不得省略：

| 序号 | 章节名称 | 内容要求 |
|-----|---------|---------|
| 1 | 产品概述 | 产品简介、核心功能一览、系统要求 |
| 2 | 安装与启动 | 安装版/便携版安装步骤、首次运行注意事项 |
| 3 | 界面布局 | 整体布局结构图、各区域功能说明 |
| 4 | 快速入门 | 从配置到导出的完整操作流程 |
| 5 | 功能详解 | 各功能模块的详细操作说明 |
| 6 | 图例说明 | 节点类型、关系类型、线型的含义说明 |
| 7 | 常见问题 | FAQ 格式的问题与解答 |
| 8 | 附录 | 完整流程图、配置参考、文件格式说明 |

### 2.2 可选章节

根据版本内容可选择性添加：

- **更新日志** — 列出本版本的新增、变更、移除功能
- **词汇表** — 解释专业术语
- **快捷操作卡片** — 可裁剪的快速参考卡

### 2.3 章节编排原则

> **规则 2.1**：章节按"从概览到细节、从入门到进阶"的逻辑编排。用户应能按顺序阅读完成入门，也可按需跳转查阅特定功能。

> **规则 2.2**：章节编号采用"章号.节号"格式（如 3.2），最多支持三级标题（如 3.2.1）。避免超过三级嵌套。

## 3 内容撰写规范

### 3.1 语言风格

> **规则 3.1**：使用简体中文撰写，面向普通用户，语言简洁、准确、友好。避免技术行话和开发术语。

**✓ 推荐写法：**

> 点击"生成分解图"按钮，AI 将自动分析权利要求并生成可视化图形。

**✗ 不推荐写法：**

> 点击按钮后，前端通过 Pinia Store 调用 AI Service 的 extractActiveClaim 方法，将请求发送至 LLM API 进行特征抽取，返回的 ExtractResult 经 edge-builder 和 node-builder 处理后渲染到 X6 Graph 实例上。

### 3.2 术语使用

| 统一术语 | 避免使用 | 说明 |
|---------|---------|------|
| 权利要求 | claim、权利项 | 专利领域的标准用语 |
| 分解图 | 结构图、关系图、graph | 本软件生成的图形统称 |
| 节点 | node、顶点 | 图形中的元素节点 |
| 边 / 关系线 | edge、连线、link | 连接节点的线 |
| 画布 | canvas、viewport | 图形编辑区域 |
| 标签页 | tab、选项卡 | 多条权利要求的切换标签 |
| API Key | 密钥、token、key | AI 服务的访问凭证 |
| 限定框 | group、分组框 | 将相关节点归组的虚线框 |

> **规则 3.2**：同一概念在全文中使用统一术语，首次出现时可用括号标注英文原文，之后不再重复标注。

### 3.3 操作描述规范

> **规则 3.3**：操作步骤使用编号列表，每步以动词开头，明确操作对象和预期结果。

**✓ 推荐写法：**

1. 点击标题栏右侧的齿轮图标，打开"API 配置"对话框
2. 在服务商标签中选择 AI 服务商
3. 输入 API Key
4. 点击"测试连接"验证配置

**✗ 不推荐写法：**

1. API 配置在标题栏可以打开
2. 选一个服务商
3. 填 key
4. 测试一下

### 3.4 界面引用规范

> **规则 3.4**：引用界面元素时，使用引号标注元素名称，并说明其所在位置。按钮名称用引号包裹，菜单路径用"→"连接。

**✓ 推荐写法：**

- 点击工具栏中的"导出"按钮，选择"导出 PNG"
- 右键点击节点 → 选择"编辑"

### 3.5 提示信息规范

说明书中使用以下三类提示框，各有明确用途：

| 类型 | 图标 | 用途 | 颜色 |
|-----|-----|-----|-----|
| 提示 | 💡 | 补充说明、使用技巧、最佳实践 | 蓝色边框 |
| 警告 | ⚠ | 重要注意事项、可能导致数据丢失的操作 | 橙色边框 |
| 成功 | ✓ | 确认操作结果、保障性说明 | 绿色边框 |

> **规则 3.5**：提示框应简明扼要，每条提示只包含一个要点。避免在提示框中放置大段操作步骤。

## 4 图示规范

### 4.1 图示类型与使用场景

| 图示类型 | 使用场景 | 示例 |
|---------|---------|------|
| 布局示意图 | 展示界面整体布局和区域划分 | 界面三栏布局图 |
| 流程图 | 展示操作步骤或数据流向 | 快速入门流程、AI 抽取流程 |
| 架构图 | 展示功能模块关系 | 核心功能架构图 |
| 界面模拟图 | 模拟界面元素的外观与交互 | 对话框示意、右键菜单示意 |
| 图例展示图 | 直观展示节点/边的视觉差异 | 节点类型颜色、关系线型 |

### 4.2 图示绘制标准

> **规则 4.1**：图示使用 SVG 内嵌绘制，确保矢量输出、缩放无损。禁止使用位图截图作为示意框图。

> **规则 4.2**：图示中的颜色必须与软件实际界面一致，使用本规范定义的标准色值（见 5.4 颜色规范）。

> **规则 4.3**：流程图方向统一为从上到下或从左到右。连接线使用直角折线或直线，箭头统一使用实心三角。

> **规则 4.4**：图示宽度不超过 480px，确保在 A4 页面中完整显示。文字字号不小于 9pt。

### 4.3 图示编号与标题

> **规则 4.5**：图示编号格式为"图 章号-序号"，如"图 3-1""图 5-2"。标题置于图示下方，居中显示，字号 9.5pt，颜色 #888。

## 5 排版规范

### 5.1 页面设置

| 项目 | 规格 |
|-----|------|
| 纸张大小 | A4（210mm × 297mm） |
| 页边距 | 上下 25mm，左右 20mm |
| 页码 | 底部居中，格式"— 页码 —"，9pt 灰色 |
| 封面 | 无页边距，无页码 |

### 5.2 字体与字号

| 元素 | 字体 | 字号 |
|-----|------|------|
| 章标题（H1） | Noto Sans CJK SC Bold | 20pt |
| 节标题（H2） | Noto Sans CJK SC Bold | 15pt |
| 小节标题（H3） | Noto Sans CJK SC Bold | 12pt |
| 正文 | Noto Sans CJK SC Regular | 11pt |
| 表格内容 | Noto Sans CJK SC Regular | 10.5pt |
| 图示标题 | Noto Sans CJK SC Regular | 9.5pt |
| 提示框 | Noto Sans CJK SC Regular | 10.5pt |
| 代码/文件名 | Consolas / Courier New | 10pt |

### 5.3 表格规范

> **规则 5.1**：表格表头使用深色背景（主色 #2d6aa6）+ 白色文字，隔行变色（偶数行浅灰背景），边框使用浅灰色。

> **规则 5.2**：表格列宽按内容自适应，避免空列或过宽列。功能对比类表格应包含"说明"列。

### 5.4 颜色规范

说明书使用的标准颜色体系，与软件界面保持一致：

| 用途 | 颜色名称 | 色值 |
|-----|---------|------|
| 主色/标题 | 深海蓝 | `#1a3a5c` |
| 强调色/链接 | 品牌蓝 | `#2d6aa6` |
| 部件节点 | 天际蓝 | `#1890FF` |
| 子系统节点 | 活力橙 | `#fa8c16` |
| 特征节点 | 生机绿 | `#52c41a` |
| 逻辑关系 | 神秘紫 | `#722ed1` |
| 正文文字 | 深灰 | `#333333` |
| 辅助文字 | 中灰 | `#666666` |
| 提示文字 | 浅灰 | `#888888` |
| 边框/分割线 | 淡灰 | `#e8e8e8` |
| 成功提示 | 成功绿 | `#52c41a` |
| 警告提示 | 警告橙 | `#fa8c16` |

## 6 版本更新清单模板

每次说明书更新时，须在文档末尾填写以下格式的更新清单：

| 字段 | 说明 | 示例 |
|-----|------|------|
| 说明书版本 | 与软件版本对应 | V0.2.0 |
| 更新日期 | 本次更新的日期 | 2026-05-26 |
| 更新人 | 负责更新的人员 | 张三 |
| 变更类型 | 新增 / 修改 / 删除 / 修正 | 修改 |
| 影响章节 | 受影响的章节编号 | 3.4, 5.3 |
| 变更说明 | 简要描述变更内容 | 更新画布操作说明，新增快捷键表格 |
| 图示更新 | 是否需要重新绘制图示 | 图 3-1 需重绘 |

> **规则 6.1**：版本更新清单按时间倒序排列，最新版本在最上方。每条记录必须完整填写所有字段。

> **规则 6.2**：大版本更新时，需在清单前增加"版本概述"段落，总结本版本的主要变化。
